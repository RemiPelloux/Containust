Containust

Containers without the daemon. Declare a stack in .ctst, pull digest-pinned images, run on Linux — or a lightweight VM on macOS and Windows.

Get started Download ctst
COMPONENT api
image  = "file:///opt/api"
port   = 8080
memory = "256MiB"
COMPONENT db
image = "tar:///opt/pg.tar"
port  = 5432

CONNECT api → db

No long-lived root process.

Each ctst command does the work, writes project state under .containust/, and exits. Offline mode fails closed before any network call. Built for machines that cannot — or must not — run Docker’s daemon.

Declare

A small grammar: components, connections, exposes, restart, and healthchecks — not a second operating system.

Verify

SHA-256 layers, digest-pinned pulls, and a content-addressed catalog you can copy between air-gapped hosts.

Isolate

Linux namespaces, cgroups v2, and read-only rootfs by default. QEMU when the host is not Linux.

One file. A whole stack.

Plan, build, run, inspect — the same verbs operators already know, without a background service holding privileges.

  1. ctst pull alpine:3.21
  2. ctst run stack.ctst --detach
  3. ctst ps --all
COMPONENT web {
    image   = "preset://alpine"
    command = ["/bin/busybox", "httpd", "-f", "-p", "8080"]
    ports   = ["8080"]
    restart = "on-failure"
}

EXPOSE 8080
