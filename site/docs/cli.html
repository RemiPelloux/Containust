Docs / CLI

ctst 1.2.0

CLI reference

One binary. No background daemon. On Linux it talks to the kernel; on macOS and Windows it uses a lightweight QEMU VM. 

ctst [OPTIONS] <COMMAND> [ARGS...]

Global options

Flag Description Env
--offline Block all outbound network during build and run CONTAINUST_OFFLINE=1
--state-file <PATH> State index path (default project-local) CONTAINUST_STATE_FILE
--help / --version Help and version —

Logging uses CONTAINUST_LOG with tracing filter directives (e.g. info, containust_runtime=debug).

Lifecycle

ctst plan

Parse and validate a composition; print deploy order without importing images.

ctst plan [FILE]
# default FILE: containust.ctst

ctst build

Import every declared image into the content-addressed catalog.

ctst build [OPTIONS] [FILE]
  --dry-run    Plan imports without writing layers

ctst run

Start the component graph. Foreground by default; use --detach for background.

ctst run [OPTIONS] [FILE]
  --detach     Run in the background
ctst run stack.ctst --detach

ctst stop / ctst rm

ctst stop <NAME|ID>
ctst rm [<NAME|ID>] [--force]

Images

ctst pull

Pull an OCI image into the local catalog and print a digest-pinned URI.

ctst pull alpine:3.21
# image://library/alpine@sha256:…

ctst images

ctst images
ctst images --presets

Inspect

Command Purpose
ctst ps [--all] List containers; reconciles restart/health policies
ctst logs <NAME> [--follow] Read detached stdout/stderr
ctst exec <NAME> -- <CMD…> Run a command inside a running container

ctst convert

Translate Docker Compose YAML into a .ctst draft.

ctst convert docker-compose.yml > stack.ctst

ctst vm

QEMU lifecycle helpers for macOS and Windows.

ctst vm start
ctst vm stop
