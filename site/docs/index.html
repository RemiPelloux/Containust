Get started with Containust

Daemon-less containers for local, sovereign, and air-gapped work. Install ctst, declare a stack in .ctst, run.

Verified release binaries, packages, or cargo.

First run

Five commands from empty host to a running container.

.ctst language

Components, connections, exposes, healthchecks.

From Docker

Map Compose YAML to Containust and convert stacks.

Mental model

  • No daemon. Each ctst command talks to the kernel (Linux) or a QEMU VM and writes state under .containust/.
  • Compositions are .ctst files. Declare COMPONENTs and CONNECT edges — a smaller grammar than Compose.
  • Images are content-addressed. Pulls land as image://name@sha256:…. Offline runs use only the local catalog.
  • Reconciliation is on demand. Restart and healthcheck policies apply when you invoke ctst ps or ctst run.

Install

Prefer a verified release binary:

VERSION=1.2.0
TARGET=x86_64-unknown-linux-gnu   # or aarch64-*-gnu, *-apple-darwin, …
curl -LO "https://github.com/RemiPelloux/Containust/releases/download/v${VERSION}/ctst-${TARGET}.tar.gz"
curl -LO "https://github.com/RemiPelloux/Containust/releases/download/v${VERSION}/ctst-${TARGET}.tar.gz.sha256"
sha256sum -c "ctst-${TARGET}.tar.gz.sha256"
tar xzf "ctst-${TARGET}.tar.gz"
sudo install -m 755 ctst /usr/local/bin/ctst
ctst --version
Channel Command
Debian / Ubuntu sudo dpkg -i ctst_1.2.0_amd64.deb
Fedora / RHEL sudo rpm -i ctst-1.2.0-1.amd64.rpm
Homebrew brew install --formula ./Formula/ctst.rb
From source cargo install --path crates/containust-cli
macOS / Windows need QEMU 7+ (brew install qemu or the Windows installer). First run downloads Alpine VM assets into ~/.containust/cache/.

First container

  1. Pull a small image into the local catalog
  2. Write a composition (or use examples/alpine_preset.ctst)
  3. Validate and import images
  4. Run, inspect, tear down
ctst pull alpine:3.21

cat > hello.ctst <<'EOF'
COMPONENT app {
    image   = "preset://alpine"
    command = ["/bin/busybox", "echo", "hello from Containust"]
}
EOF

ctst plan hello.ctst
ctst build hello.ctst
ctst run hello.ctst
# or: ctst run hello.ctst --detach

ctst ps --all
ctst logs app
ctst stop app && ctst rm app

preset://alpine downloads a pinned Alpine minirootfs (~4 MiB) on first use. Use ctst pull for arbitrary Docker Hub / GHCR images.

Air-gapped / offline

# Networked machine
ctst pull alpine:3.21
ctst build stack.ctst
# Copy .containust/images and .containust/layers

# Air-gapped host
ctst --offline run stack.ctst --detach

--offline (or CONTAINUST_OFFLINE=1) rejects any remote fetch before connecting.

Multi-service sketch

COMPONENT db {
    image   = "image://library/postgres@sha256:…"
    port    = 5432
    env     = { POSTGRES_PASSWORD = "secret" }
    volumes = ["/data/pg:/var/lib/postgresql/data"]
    restart = "always"
}

COMPONENT api {
    image   = "file:///opt/images/api"
    port    = 8080
    ports   = ["8080"]
    memory  = "256MiB"
    restart = "on-failure"
}

CONNECT api -> db
EXPOSE 8080

CONNECT injects DB_HOST, DB_PORT, and related env vars into api. Volumes are host:container bind mounts.

Troubleshooting

Symptom Try
offline mode rejects remote source Drop --offline, or pre-pull / copy catalog layers
Port bind fails Free the host port; stop conflicting forwarders / QEMU hostfwd
Container failed after --detach ctst logs <name> — binary missing or command exited
Permission denied on Linux Confirm cgroup v2 and namespace privileges
Private registry 401 Set CONTAINUST_REGISTRY_* or refresh Docker config
