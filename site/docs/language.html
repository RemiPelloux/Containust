Composition language

The .ctst format

Declarative, LLM-friendly, statically analyzable. Uppercase keywords, local-first image sources, secure defaults.

Annotated example

IMPORT "templates/postgres.ctst" AS pg

COMPONENT api {
    image   = "file:///opt/images/myapp-api"
    port    = 8080
    memory  = "256MiB"
    cpu     = "1024"
    env     = {
        RUST_LOG     = "info"
        DATABASE_URL = "postgres://${db.host}:${db.port}/app"
    }
    command  = ["./server", "--bind", "0.0.0.0:8080"]
    readonly = true
}

COMPONENT db FROM pg {
    port   = 5432
    volume = "/data/pg:/var/lib/postgresql/data"
    env    = { POSTGRES_PASSWORD = "${secret.db_pass}" }
}

CONNECT api -> db
EXPOSE 8080
ctst plan stack.ctst
ctst build stack.ctst
ctst run stack.ctst --detach

Design principles

Principle Why
Declarative Describe what you want; the runtime orders and wires
LLM-friendly Minimal syntax, consistent structure, uppercase keywords
Statically analyzable Types, references, cycles, unused imports before create
Local-first file:// / tar:// preferred; remote is opt-in
Secure by default Read-only rootfs, dropped capabilities, secret isolation

COMPONENT

Each service is a COMPONENT block. Common fields:

Field Example
image "preset://alpine", "file:///…", "image://…@sha256:…"
port / ports 8080 · ["8080", "8443"]
memory / cpu "256MiB" · "1024" (shares)
env { KEY = "value" }
command / entrypoint ["./server", "--bind", "0.0.0.0:8080"]
volumes ["/host/data:/var/lib/data"]
restart "never" · "on-failure" · "always"
readonly Default true
network "none" (default) · "bridge" · named net

CONNECT

CONNECT api -> db means api depends on db. Containust starts db first and injects DB_HOST, DB_PORT, and DB_CONNECTION_STRING into api.

EXPOSE

EXPOSE 8080          # identity publish
EXPOSE 80:8080       # host 80 → container 8080
  • Linux (identity, no network): host netns; bind directly
  • Linux (remap / named network): private or shared netns + userspace forwarder
  • macOS / Windows: QEMU hostfwd

Healthcheck

COMPONENT api {
    image = "image://library/myapi@sha256:…"
    port  = 8080
    healthcheck = {
        command      = ["wget", "-qO-", "http://127.0.0.1:8080/health"]
        interval     = "10s"
        timeout      = "3s"
        retries      = 3
        start_period = "5s"
    }
    restart = "on-failure"
}

Policies are enforced when you run ctst ps or ctst run — there is no background supervisor daemon.

IMPORT / FROM

Reuse blocks with IMPORT "path.ctst" AS alias and COMPONENT name FROM alias { … } to override fields.
